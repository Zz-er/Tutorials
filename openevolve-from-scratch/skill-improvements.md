# reimpl-tutorial Skill 改进经验记录

> 在制作 OpenEvolve 教程过程中积累的经验，后续统一更新到 skill 中。

## 1. JSON 格式问题（关键 Bug）

**问题：** 使用 Write 工具直接写 .ipynb 文件时，中文文本中的双引号（如 "更好"）会导致 JSON 解析失败。

**解决方案：**
- 应使用 Node.js 脚本程序化构建 notebook，通过 `JSON.stringify()` 保证 JSON 合法性
- 或者在 skill 中明确要求：中文文本使用「」代替 ""，避免 JSON 转义问题
- 建议在 skill 中添加一个验证步骤：每写完一个 notebook 必须用 `JSON.parse()` 验证

**改进建议：** 在 SKILL.md 中添加 "Notebook 质量检查" 步骤，要求每个 notebook 写完后执行 JSON 验证。

## 2. 中文引号规范

**问题：** 中文中习惯使用 "" 但这些字符在 JSON 字符串中虽然合法，在某些场景可能引起混淆。

**建议：** 在 skill 的 notebook 标准中建议使用「」作为中文引号，或者确保所有 .ipynb 通过脚本生成。

## 3. 贯穿例子的连贯性

**经验：** 排序算法作为贯穿例子效果很好——每章都能自然引入新概念：
- 第一章：冒泡排序（最简单）
- 第二章：多种排序算法共存（MAP-Elites 的多样性）
- 第三章：不同岛屿独立进化不同排序方法

**改进建议：** 在 skill 中强化"贯穿例子"的选择标准，要求例子必须能在每个层级自然扩展。

## 4. 代码可运行性

**经验：** Notebook 中的代码引用了 `our-implementation/core.py` 等文件，这些文件是由前面章节的"保存模块"代码单元创建的。需要确保：
- 每章的代码在该章内部自包含（可以独立运行）
- 或者明确说明依赖前面章节的输出

**改进建议：** 在 skill 中添加两种模式：
1. 自包含模式：每个 notebook 完全独立
2. 递进模式：后面章节依赖前面章节的产出（当前采用的方式）

## 5. 并行 Agent 策略

**经验：** 并行启动多个 agent 创建 notebook 容易触发 API 速率限制。

**改进建议：** 在 skill 中建议串行创建 notebook，避免并行导致的速率限制问题。

## 6. 数学公式的渲染兼容性

**经验：** LaTeX 公式在 Jupyter 中需要使用 `$...$` 和 `$$...$$`，在 JSON 中反斜杠需要双重转义（`\\`）。

**改进建议：** 在 skill 的模板中提供 LaTeX 转义示例，避免常见的转义错误。

## 7. 对照源码表格

**经验：** "我们的实现 vs. OpenEvolve 源码"的对照表非常有教学价值，读者可以直接对照阅读。

**改进建议：** 在 skill 的 feature-template 中将"对照源码映射表"作为必须包含的部分。

## 8. Mermaid 图表

**经验：** Mermaid 图在 Jupyter 中需要作为 markdown 单元格中的代码块呈现，不能放在代码单元格中。

**改进建议：** 在 skill 中明确说明 mermaid 图的使用限制和正确格式。

## 9. 公式解读优化——提供通俗易懂的例子

**问题：** 数学公式（如 MAP-Elites 适应度函数、迁移概率公式）仅给出 LaTeX 定义时，零基础读者仍然难以理解其含义。

**改进方案：** 每出现一个公式，都应紧跟一个**具体数值例子**来解释：

1. **先给公式**
   $$fitness = \frac{1}{K}\sum_{k=1}^{K} m_k$$

2. **再给例子**
   > 假设排序算法在三个指标上得分：正确性=0.9，速度=0.7，内存=0.8，那么：
   > $fitness = \frac{1}{3}(0.9 + 0.7 + 0.8) = 0.8$
   > 就像期末考试算平均分——把每科成绩加起来除以科目数。

3. **生活类比**
   > 用一句生活中的类比帮助读者建立直觉（如「期末考试平均分」「抽奖轮盘」「邮局分拣信件」）。

**改进建议：** 在 skill 的 `feature-template.md` 和 `derivation-prompt.md` 中添加强制要求：
- 每个公式后必须附带一个**具体数值计算例子**
- 每个公式后建议附带一个**生活类比**
- 例子中的数值应来自教程中的贯穿例子（排序算法），保持一致性

**模板示例（供 skill 使用）：**
```
### 公式：[公式名称]

$$[LaTeX 公式]$$

**通俗解释：** [一句话说明公式做了什么]

**数值例子：**
> [用具体数字代入公式，展示计算过程和结果]

**生活类比：**
> [用日常经验类比，帮助建立直觉]
```
